Rebase

		       git rebase  -i revision
		       git rebase  -i branch
		       There is a way for using merge --rebase, pull --rebase

Workflow

  • Branching
    1. One remote
    2. Each task/subtask/but in other branch
    3. Master is release branch
  • Using forks
    1. Fork per developer
    2. Each task/subtask/but in other branch
    3. Master is release branch
    4. Send pull requests
